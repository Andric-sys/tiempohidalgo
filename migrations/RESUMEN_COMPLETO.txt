## 🗄️ Tablas Creadas (7 totales)

### 1. USUARIOS
```
- id (PK)
- usuario (UNIQUE)
- contrasena (MD5)
- email
- estado (activo/inactivo)
- fecha_creacion
- fecha_actualizacion
```
**Índices:** usuario, estado

### 2. NOTICIAS
```
- id (PK)
- titulo
- descripcion
- estado (borrador/publicado/archivado)
- autor_id (FK → usuarios)
- fecha_creacion
- fecha_actualizacion
- fecha_publicacion
```
**Índices:** fecha_creacion, estado, autor_id

### 3. BLOQUES (Sistema flexible)
```
- id (PK)
- noticia_id (FK → noticias)
- tipo (imagen/parrafo/titulo/subtitulo)
- contenido (LONGTEXT)
- orden (1, 2, 3...)
- fecha_creacion
```
**Índices:** noticia_id, tipo, orden

### 4. GALERIA
```
- id (PK)
- noticia_id (FK → noticias, opcional)
- titulo
- imagen
- descripcion
- orden
- fecha_creacion
```
**Índices:** noticia_id, orden

### 5. CATEGORIAS
```
- id (PK)
- nombre (UNIQUE)
- descripcion
- slug (UNIQUE)
- fecha_creacion
```
**Índices:** slug

### 6. NOTICIA_CATEGORIA (Relación M:N)
```
- noticia_id (PK+FK → noticias)
- categoria_id (PK+FK → categorias)
```

### 7. CONTACTOS
```
- id (PK)
- nombre
- email
- telefono
- asunto
- mensaje
- estado (nuevo/respondido/cerrado)
- fecha_creacion
```
**Índices:** estado, fecha_creacion

## 🎓 Concepto de Bloques

Un bloque es un contenedor flexible para:
- **Imagen:** Referencia a archivo subido
- **Párrafo:** Texto del artículo
- **Título:** Encabezado de sección
- **Subtítulo:** Subtítulo de sección

Cada noticia puede tener N bloques en cualquier orden.

**Ejemplo:**
```
Noticia: "Noticias Locales"
├─ Bloque 1: Imagen (header.jpg)
├─ Bloque 2: Párrafo (Introducción...)
├─ Bloque 3: Imagen (foto1.jpg)
├─ Bloque 4: Párrafo (Desarrollo...)
└─ Bloque 5: Imagen (foto2.jpg)
```
